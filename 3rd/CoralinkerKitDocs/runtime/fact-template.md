# fact

记录已经向用户确认过的现场硬件、车辆、协议、安全和应用事实。Agent 第一次执行真实硬件任务时，应把本模板复制或整理到：

```text
ai-deck/fact.md
```

本文件只写已确认事实、来源和待确认问题，不写猜测。

## 设备与节点

- 待确认：MCU 节点数量、名称、UUID、物理位置。
- 待确认：每个节点连接的执行器、传感器、灯、按键、急停、触边、防撞、限位、安全继电器。

## 接线与端口

- 待确认：每个设备接在哪个节点、端口、通道。
- 待确认：IO 极性、电平、默认状态、上电状态。
- 待确认：串口/CAN/IO 参数，例如波特率、CAN ID、payload 格式、字节序、缩放系数。

## 机械与运动

- 待确认：车辆构型，例如差速、前后桥舵轮、四舵轮、麦轮、履带、叉车式。
- 待确认：安装位置、轮径、减速比、负载重量。
- 待确认：最大速度、最大加速度、最大舵角、机械限位、控制方向正负约定。

## 控制协议

- 待确认：驱动器/传感器型号。
- 待确认：控制协议、寄存器/对象字典、命令格式、反馈格式、故障码定义。

## 安全策略

- 待确认：急停、触边、防撞、限位触发后的动作。
- 待确认：通信断开多久进入安全状态。
- 待确认：是否允许自动复位故障，还是必须人工确认。
- 待确认：真实硬件启动授权模式：人类手动 Start、每次确认后 Agent Start、或本次会话授权 Agent 直接 Start。

## 应用场景与节拍

- 待确认：实际应用场景、作业流程、运行节拍、人机协作边界。
- 待确认：验证方式，例如 Variables、节点日志、WireTap、实际轮子动作、模拟节点测试。
